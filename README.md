## Лабораторная работа №1 — Цифровое представление изображения

Что реализовано:
- Загрузка изображений через кнопку и drag and drop
- Поддержка форматов PNG, JPG/JPEG и собственного формата GB7
- Отображение на HTML5 Canvas
- Сохранение в PNG, JPG, GB7
- Кодер и декодер формата GrayBit-7 написан вручную без библиотек
- Строка состояния: имя файла, формат, размер, глубина цвета, мегапиксели
- Адаптивная вёрстка под разные размеры экрана

## Лабораторная работа №2 — Цветовые каналы и инструмент Пипетка

Что реализовано:
- Панель каналов с миниатюрами, автоматически определяет тип изображения:
  grayscale (1 канал), grayscale + alpha (2), RGB (3), RGB + alpha (4)
- Каждый канал отображается в оттенках серого (интенсивность канала)
- Клик по каналу включает или выключает его отображение на холсте
- При отключении всех каналов изображение корректно скрывается (становится чёрным)
- Если активен только Alpha — показывается маска прозрачности
- Оригинальные данные изображения хранятся отдельно и не перезаписываются
- Инструмент Пипетка: активируется кнопкой в тулбаре
- При клике по изображению показывает: координаты X/Y, значения R G B (0–255),
  HEX-код, значение Alpha, координаты в цветовом пространстве CIELAB (L* a* b*)
- Конвертация sRGB в CIELAB реализована вручную: sRGB → линейный RGB → XYZ → Lab
- Масштабный коэффициент canvas учитывается при пересчёте координат клика

## Лабораторная работа №3 — Градационные преобразования и инструмент «Уровни»

Что реализовано:
- Гистограмма изображения: подсчёт яркости по всем пикселям (`(76R + 150G + 30B) >> 8`), отрисовка столбчатой диаграммы на `<canvas>`, переключение между линейной и логарифмической шкалой по оси Y. При смене канала гистограмма перестраивается мгновенно
- Диалог открывается через нативный `<dialog>` (showModal). Три маркера на слайдере — чёрная точка, гамма (0.1–9.9), белая точка — с поддержкой перетаскивания мышью и тач-событиями. Маркеры не пересекаются
- Коррекция применяется к Master и каждому каналу (R, G, B, Alpha) независимо, настройки сохраняются при переключении. LUT строится как композиция Master + канал, применяется за один проход
- Предпросмотр обновляется в реальном времени через requestAnimationFrame (троттлинг). Чекбокс позволяет мгновенно сравнить до/после. Кнопки: сброс канала, сброс всего, отмена, применить. Оригинальные данные не изменяются до нажатия «Применить»

## Лабораторная работа №4 — Масштабирование изображения

Что реализовано:
- Два метода двумерной интерполяции написаны вручную без библиотек: **ближайший сосед** — копирует ближайший пиксель без смешивания; **билинейная** (по умолчанию) — усредняет 4 соседних пикселя с весами по осям X и Y. Единая точка входа `scaleImage()` через реестр `INTERPOLATION_METHODS` — новый метод добавляется одной записью в объект
- В информационной панели размещён `ScaleSelector`: выпадающий список пресетов (12 / 25 / 33 / 50 / 67 / 75 / 100 / 150 / 200 / 300 %), ползунок `range` 12–300 % с RAF-троттлингом, кнопки «−» / «+» для перехода по пресетам. При загрузке масштаб вычисляется автоматически через `fitScale()` — изображение вписывается в canvas с отступом 50 px, не выходя за диапазон 12–300 %
- Инструмент изменения размера (`ResizeDialog`): мегапиксели до и после (пересчёт на лету), единицы px/% с автоконвертацией, связь пропорций через чекбокс, выбор метода интерполяции с тултипом-описанием, валидация ввода (целые числа, px: 1–16 000, %: 1–1 000)
- Все тяжёлые операции (интерполяция, LUT, гистограммы) вынесены в Web Worker (`imageProcessor.worker.js`) — главный поток не блокируется, данные передаются zero-copy через `Transferable`. Токен `opTokenRef` в `useCanvas.js` отменяет устаревшие async-результаты при быстром вводе
- Исправлен баг пипетки: координаты клика пересчитывались с двойным масштабом — лишний множитель убран, теперь показывается точный пиксель оригинала на любом масштабе
- Добавлено масштабирование колесом мыши при зажатом Ctrl

## Лабораторная работа №5 — Фильтрация ядром свёртки

Что реализовано:
- Диалог `KernelDialog` открывается через нативный `<dialog>`. Кнопки: **Закрыть**, **Сброс**, **Применить**; чекбокс **Предпросмотр**
- Сетка 3×3 из 9 редактируемых полей. Поддерживаются дроби (`1/16`), отрицательные числа; ошибочные ячейки подсвечиваются. При ручном изменении пресет переключается в «пользовательское»
- Шесть пресетов — выбор заполняет поля, значения можно менять вручную:
  1. Тождественное отображение
  2. Повышение резкости
  3. Фильтр Гаусса 3×3
  4. Прямоугольное размытие (Box Blur)
  5. Оператор Прюитта — горизонтальный
  6. Оператор Прюитта — вертикальный
- Чекбоксы каналов: R, G, B, Alpha — применить можно к любому набору, снять последний нельзя
- Три стратегии обработки края: заполнение чёрным, белым, копирование крайнего пикселя (`replicate`). Размерность изображения не уменьшается
- Предпросмотр работает на уменьшенной копии ≤300px через Web Worker с RAF-троттлингом — интерфейс не блокируется. Если ядро тождественное — Worker не запускается
- Применение к полному изображению в Web Worker с прогресс-баром (0–100%)
- Оптимизация свёртки: центральная часть (99%+ пикселей) обрабатывается без единой проверки границ и без вызовов функций, ядро развёрнуто в 9 переменных — ускорение ~15× по сравнению с наивной реализацией

## Формат GB7

Бинарный формат для хранения изображений в оттенках серого.

Заголовок (12 байт):
- Байты 0–3: сигнатура `47 42 37 1D`
- Байт 4: версия (`0x01`)
- Байт 5: флаги (бит 0 — маска прозрачности)
- Байты 6–7: ширина (big-endian)
- Байты 8–9: высота (big-endian)
- Байты 10–11: зарезервировано

Каждый пиксель — один байт: биты 6–0 — яркость (0–127), бит 7 — маска.
